# Phase 0 Plan: Setup + Baseline

**Goal:** Get your environment working, understand the curriculum, and build the habits that will carry through the rest of the program.

---

## What to do

### Step 1 — Read the curriculum
- Read `docs/CURRICULUM-MAP.md` end-to-end. Understand the phase structure and dependency order.
- Read `docs/PLAN.md` for the full intent behind the program.
- You should be able to answer: what are the 6 phases? What does each one build toward?

### Step 2 — Environment setup
- Confirm your terminal is working: open it, run `echo "hello"`
- Confirm Git is installed: `git --version`
- Confirm VS Code (or your editor) is installed and can open this repo
- Confirm you can push to this repo: check `git remote -v`

### Step 3 — Start the glossary habit
- Open `docs/GLOSSARY.md`
- Add 3–5 terms you already know and want to anchor (e.g. API, database, repository)
- This is a living document — add to it at the end of every module

### Step 4 — First log entry
- Open `docs/LOG.md`
- Write an entry: what you set up, what you noticed, what you're curious about going into Phase 1

---

## Phase 0 is complete when:
- [ ] Terminal, Git, and editor all confirmed working
- [ ] `docs/GLOSSARY.md` has at least 5 entries you wrote
- [ ] First log entry committed to `docs/LOG.md`
- [ ] You can describe the 6 phases and their dependency order without looking

**Next:** Open `docs/phases/PHASE-1.md`
